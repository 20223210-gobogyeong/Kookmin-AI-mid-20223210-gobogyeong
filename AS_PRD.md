# 수정된 PRD: Arch-Sync (개인용 대시보드)
(기술 제약: HTML, CSS, JS + localStorage, No-Backend)

## 1. 개요 및 목표

- **제품명:** Arch-Sync (개인용)
- **수정된 목표:** 개인이 설계 스튜디오 프로젝트를 진행할 때, 자신의 할 일, 중요 일정, 리서치 **링크**를 한곳에 모아 체계적으로 관리하고 진행 상황을 놓치지 않도록 돕는다.
- **핵심 전제:** **팀 데이터 공유 기능이 없는** 개인 전용 툴이다.

## 2. 대상 사용자 (Personas)

- **김지수 (팀장, 3학년):** "팀 프로젝트가 복잡하지만, 일단 **나에게 할당된 일**과 내가 챙겨야 할 **중요 일정**부터 놓치지 않고 관리하고 싶다."
- **이철민 (팀원, 3학년):** "리서치한 사이트, 구글 드라이브 링크, 참고할 이미지 URL이 흩어져 있는데, 프로젝트별로 깔끔하게 **링크를 저장**해두고 싶다."

## 3. 핵심 기능 (Features) - LocalStorage 기준

### F-01: 대시보드 (메인 페이지)
- **설명:** 사용자가 앱을 열었을 때 가장 먼저 보이는 개인화된 요약 화면.
- **구현 요소:**
    - `[Events]` 데이터와 연동된 "다가오는 중요 일정" (예: D-3: 2차 크리틱)
    - `[Tasks]` 데이터와 연동된 "오늘 내가 할 일 목록"
    - `[Feeds]` 데이터와 연동된 "최근 내 메모/피드백"

### F-02: 리소스 아카이브 (링크 관리)
- **설명:** 실제 파일을 업로드하는 대신, 프로젝트 관련 URL(구글 드라이브, 핀터레스트, 레퍼런스 사이트 등)을 저장하고 관리하는 공간.
- **사용자 스토리:** "팀원은 '패널' 카테고리에 'v3_최종본_구글드라이브_링크'를 저장해두고, 필요할 때 바로 클릭해서 방문할 수 있어야 한다."
- **구현 요소:**
    - `[Resources]` 데이터 생성 및 리스트 뷰
    - **URL 링크 추가 기능** (파일 업로드 X)
    - 카테고리별 필터링 기능 (도면 링크, 패널 링크, 리서치 사이트)

### F-03: 칸반 스타일 할 일 관리 (개인용)
- **설명:** **자신의** 모든 할 일을 시각적으로 추적하고 관리한다.
- **사용자 스토리:** "팀장은 '모델 제작'이라는 태스크를 생성하고, '담당자' 필드에 단순히 텍스트로 '이철민'이라고 **기록**해 둘 수 있다. 이 작업의 상태를 '진행 중', '완료'로 변경하며 개인적으로 진행률을 체크한다."
- **구현 요소:**
    - `[Tasks]` 데이터 생성 및 칸반 뷰(Kanban View)
    - 상태 필드(대기, 진행 중, 완료)를 기준으로 드래그 앤 드롭 인터페이스
    - '담당자'는 다른 사용자에게 할당(Assign)되는 기능이 아닌, 단순 **텍스트 입력 필드**여야 함.

### F-04: 공지 및 피드백 노트 (개인 메모장)
- **설명:** 교수님 크리틱 피드백, 회의록 등 **개인이 기억해야 할 내용**을 기록하는 개인 메모장.
- **사용자 스토리:** "팀원은 지난주 교수님 크리틱 때 나왔던 '매스 변경' 피드백을 잊지 않기 위해 여기에 기록해두고, 패널 작업 시 다시 찾아볼 수 있어야 한다."
- **구현 요소:**
    - `[Feeds]` 데이터 생성 및 블로그형 리스트 뷰
    - 유형별(피드백, 회의록, 아이디어) 필터링 기능

## 4. 데이터 구조 (LocalStorage Schema)

`localStorage`는 "테이블"이 아닌 "Key-Value" 저장소이다. 모든 데이터는 **JSON 문자열**로 변환되어 저장되어야 한다. (예: `localStorage.setItem('archsync_tasks', JSON.stringify(tasksArray))`)

| localStorage Key | 데이터 타입 | 설명 |
| :--- | :--- | :--- | :--- |
| `archsync_userProfile` | `Object` | 사용자의 이름 등 간단한 프로필. (예: `{ "name": "김지수" }`) |
| `archsync_tasks` | `Array<Object>` | 할 일 목록. (아래 Tasks 객체 구조 참고) |
| `archsync_resources` | `Array<Object>` | 리소스(링크) 목록. (아래 Resources 객체 구조 참고) |
| `archsync_feeds` | `Array<Object>` | 개인 메모 목록. (아래 Feeds 객체 구조 참고) |
| `archsync_events` | `Array<Object>` | 일정 목록. (아래 Events 객체 구조 참고) |

---

#### 객체 상세 구조 (예시)

- **Tasks 객체:**
  ```json
  {
    "id": "t1",
    "taskName": "모델 제작",
    "assigneeName": "이철민 (메모용)", 
    "status": "대기",
    "dueDate": "2025-11-10"
  }
  ```
- **Resources (Files) 객체:**
  ```json
  {
    "id": "r1",
    "resourceName": "v3 최종 패널",
    "url": "[https://docs.google.com/](https://docs.google.com/)...", 
    "category": "패널 링크",
    "registrant": "김지수 (메모용)",
    "version": "v3"
  }
  ```
- **Feeds 객체:**
  ```json
  {
    "id": "f1",
    "title": "2차 크리틱 피드백",
    "content": "매스 재검토 필요...",
    "type": "피드백",
    "authorName": "김지수 (메역용)"
  }
  ```
- **Events 객체:**
  ```json
  {
    "id": "e1",
    "eventName": "최종 마감",
    "date": "2025-11-15"
  }
  ```

## 5. 범위 외 기능 (Out-of-Scope)

- **실시간 팀 데이터 동기화 (기술적 제약으로 절대 불가)**
- **사용자 로그인 및 계정 시스템 (불가)**
- **실제 파일 업로드 (`.dwg`, `.ai` 등) (불가 - 대신 링크 저장으로 대체)**
- 실시간 채팅 기능
- 복잡한 간트 차트
